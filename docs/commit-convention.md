# Commit Convention

This repository uses Conventional Commits.

## Format

```text
<type>: <summary>
```

Use a short imperative summary in English.

## Types

- `feat`: introduce a new user-facing or project-facing capability.
- `fix`: correct broken behavior.
- `docs`: update documentation only.
- `refactor`: restructure without changing intended behavior.
- `chore`: maintain tooling, repository metadata, or housekeeping.
- `test`: add or update tests.
- `ci`: update automation or CI configuration.

## Examples

```text
feat: add feature lifecycle workflow
docs: describe PBDD conformance rules
refactor: separate runtime references from skill instructions
chore: configure repository line endings
```
